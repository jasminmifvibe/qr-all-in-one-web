# QR All-in-One Static Site

Simple static GitHub Pages site for the iOS app **QR All-in-One**.

## Files

- `index.html` - homepage
- `support/index.html` - support page
- `privacy/index.html` - privacy policy page
- `styles.css` - shared styling

## Publish with GitHub Pages

1. Create a new GitHub repository.
2. Upload all files and folders from this project to the root of the repository.
3. Push to the `main` branch.
4. In GitHub, open **Settings** -> **Pages**.
5. Under **Build and deployment**, set:
   - **Source**: Deploy from a branch
   - **Branch**: `main`
   - **Folder**: `/ (root)`
6. Save.
7. Wait for GitHub Pages to publish the site.

## Example URLs

If your repo is named `qr-all-in-one`, your URLs will look like:

- `https://USERNAME.github.io/qr-all-in-one/`
- `https://USERNAME.github.io/qr-all-in-one/support/`
- `https://USERNAME.github.io/qr-all-in-one/privacy/`
